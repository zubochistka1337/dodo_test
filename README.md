# Анализ занятости столиков

### 🚀 Запуск проекта
Для установки зависимостей и запуска обработки видео выполните команду:
```bash
uv sync && uv run ./src/main.py --video "./видео 1.mp4"
```

### 📺 Демонстрация
* **Входные данные:** `видео 1.mp4`
* **Результат:** выбранный столик можно увидеть на `output.mp4`

### 🧠 Логика детекции событий
* **Алгоритм:** если нижняя часть bbox тела человека (1/3) имеет пороговое пересечение с bbox стола, то считаем что столик занят. Аналогично обратно, если никаких пересечений нет, считаем, что столик свободный. 
* **Обоснование:** такой подход выбран из-за 2D ракурса камеры и чтобы условно "голова" человека на переднем плане не мешала столикам на удалении. 
* **Стабилизация:** поверх этой логики накидан буффер кадров смягчающий ложные детекции, чтоб не было постоянных смен состояний, который можно настроить (по умолчанию 20 кадров). 
* **Временной фильтр:** добавлена логика, что если пересечение человека со столом длится в течении 15 секунд (по умолчанию) + смягчающий буфер кадров, то считаем это сменой состояния. 

> [!NOTE]
> Алгоритм смягчения не идеальный, рядом приложен файл с алгоритмом от нейронки, где добавлена более мягкая логика буферных кадров для смягчения ложных детекций, добавлять к себе уже не стал.

### 🛠 Спецификации
* **Модель:** Базовая `yolo11s`
* **Тестовое железо:** `r7 3700x / rtx 5080`
* **Результат:** данные сохранены в файле `avg_data.json`
